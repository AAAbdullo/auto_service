# 📱 Запуск приложения в эмуляторе iOS/Android

## 🎯 Проблема: Маршруты не строятся

Да, **маршруты не работают в эмуляторе по нескольким причинам**:

### ❌ Основные проблемы в эмуляторе:

1. **GPS/Геолокация отключена или не сконфигурирована**
   - Эмулятор не имеет реального GPS
   - Нужна либо ручная симуляция, либо fallback на тестовые координаты

2. **Разрешения на геолокацию**
   - Даже если включить, могут быть проблемы с запросом разрешений

3. **Yandex MapKit требует интернета**
   - Эмулятор должен иметь доступ в интернет
   - API ключ должен быть корректным

---

## ✅ Решение: Автоматический fallback на Ташкент

**Обновлено в версии:** `services_map_screen.dart`

Теперь приложение автоматически использует **тестовые координаты Ташкента** (41.2995, 69.2401) если:
- GPS отключен/недоступен
- Разрешения на геолокацию не даны
- Произойдет любая ошибка при получении местоположения

### 📍 Что это означает:

```
✅ Маршруты БУДУТ работать в эмуляторе
   - Стартовая точка: Ташкент (41.2995, 69.2401)
   - Вы можете выбрать любой автосервис на карте
   - Приложение построит маршрут от Ташкента до выбранного сервиса

⚠️ Вы не сможете видеть маршрут ВАС в реальное время
   - Но все маршруты будут правильно рассчитаны и отображены
```

## 🚀 Как запустить на физическом устройстве

На реальном телефоне маршруты будут использовать вашу **реальную геолокацию**:

### iOS:
```bash
flutter run
# Разрешите приложению доступ к геолокации в системных параметрах
```

### Android:
```bash
flutter run
# Разрешите приложение в диалоге разрешений
# Убедитесь, что GPS включен на устройстве
```

## 🔧 Если хотите тестировать с определенной локацией в эмуляторе

### iOS Simulator:
1. Запустите приложение
2. Откройте Xcode
3. Features → Location → City Bicycle Ride (или другой вариант)

### Android Emulator:
1. Откройте меню эмулятора (три точки справа)
2. Extended controls → Location
3. Установите широту/долготу вручную

## 🐛 Если маршруты все еще не работают

### Проверьте:

1. **Интернет в эмуляторе**
   ```bash
   # Проверьте в эмуляторе что интернет работает
   # Откройте браузер → перейдите на google.com
   ```

2. **API ключ Yandex MapKit**
   - Проверьте в `ios/Runner/Info.plist` (iOS)
   - Проверьте в `android/app/src/main/AndroidManifest.xml` (Android)
   - Убедитесь что ключ активен на сервере Яндекса

3. **Логи приложения**
   ```bash
   flutter run -v  # Режим verbose
   # Смотрите консоль на предмет ошибок "❌" и "⚠️"
   ```

4. **Проверка логов в debug-консоли**
   - Ищите сообщения вида:
   ```
   ✅ User location marker added at 41.2995, 69.2401
   ✅ Driving route found
   ⚠️ Location services disabled - using Tashkent as fallback
   ```

## 📝 Что изменилось в коде

### В `services_map_screen.dart`:

#### Было (старое поведение):
```dart
Future<Point?> _getUserLocation() async {
  // Если что-то пошло не так → null
  // Маршруты не строились!
  return null;
}
```

#### Стало (новое поведение):
```dart
Future<Point?> _getUserLocation() async {
  // Пытаемся получить реальную геолокацию
  // Если не получается → используем Ташкент как fallback
  // Маршруты ВСЕГДА строятся!
  return const Point(latitude: 41.2995, longitude: 69.2401);
}
```
